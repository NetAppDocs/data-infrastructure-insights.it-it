= Red Hat OpenShift Virtualization raccoglitore dati
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Il collector DII Red Hat OpenShift Virtualization tenterà di rilevare i carichi di lavoro di elaborazione OSV in esecuzione su un ambiente OpenShift Virtualization - questo collector non rileverà tutti i pod OpenShift. Si tratta di un collector basato su API REST HTTPS. Si consiglia di creare un collector per cluster OSV.

L'approccio di autenticazione per questo collector è un token di autenticazione associato a un account di servizio. Per creare sia l'account utente che il token:

....
oc create sa acq-readonly-cluster-user -n default
oc adm policy add-cluster-role-to-user cluster-reader -z acq-readonly-cluster-user --rolebinding-name acq-readonly-cluster-user-binding
oc adm policy add-cluster-role-to-user cluster-monitoring-view -z acq-readonly-cluster-user --rolebinding-name acq-readonly-cluster-monitor-user-binding
....
Esegui questi 3 comandi e verifica la presenza di eventuali errori.

Al termine dei comandi, esegui:

 oc create token acq-readonly-cluster-user --duration=8760h
Questo creerà un token associato al tuo utente. Nota: si consiglia di impostare la durata del token. Tieni presente che la raccolta dati non andrà a buon fine con un errore "Impossibile connettersi" dopo la scadenza del token.

Infine, potresti anche voler far eseguire al tuo amministratore OSV:

 oc cluster-info
Questo restituirà il nome host/indirizzo IP per la gestione. Ogni collector DII OSV deve essere indirizzato all'IP/nome host per la gestione OSV.



[[terminology]]
== Terminologia

Per ogni tipo di risorsa acquisita da Data Infrastructure Insights, viene mostrata la terminologia più comune utilizzata per la risorsa.  Quando si visualizza o si risolve un problema con questo strumento di raccolta dati, tenere presente la seguente terminologia:

[cols="50,50"]
|===
| Termine del fornitore/modello | Termine Data Infrastructure Insights 


| Disco | Disco virtuale 


| Ospite | Ospite 


| Macchina virtuale | Macchina virtuale 


| Dominio di archiviazione | Archivio dati 


| Unità logica | LUN 
|===
Nota: queste sono solo mappature terminologiche comuni e potrebbero non rappresentare tutti i casi per questo raccoglitore di dati.



[[requirements]]
== Requisiti

* Indirizzo IP del server Red Hat OpenShift Virtualization sulla porta 6443 tramite REST API
* Nome utente e password di sola lettura
* RHEV versione 3.0+




[[configuration]]
== Configurazione

[cols="50,50"]
|===
| Campo | Descrizione 


| Indirizzo IP del server RHEV | Indirizzo IP del sistema di archiviazione 


| Nome utente | Nome utente con privilegi di amministratore 


| Password per l'account con privilegi di amministratore | Password 
|===


[[advanced-configuration]]
== Configurazione avanzata

[cols="50,50"]
|===
| Campo | Descrizione 


| Porta di comunicazione HTTPS | Porta utilizzata per la comunicazione HTTPS con RHOV. Predefinito: 6443. 


| Intervallo di sondaggio dell'inventario (min) | Il valore predefinito è 20 minuti. 
|===
